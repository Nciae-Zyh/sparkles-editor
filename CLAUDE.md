## Project Overview

**Sparkles-Editor** is a Notion-like WYSIWYG editor built with Nuxt 4 + Vue 3. It features a TipTap-based rich text editor, document management with folder hierarchies, AI writing assistance, and is deployed on Cloudflare Workers with D1 (SQLite) and R2 (object storage).

## Commands

```bash
pnpm dev          # Start dev server at http://localhost:3000
pnpm build        # Production build (uses 8GB Node heap)
pnpm preview      # Preview production build locally
pnpm lint         # Run ESLint
pnpm lint:fix     # Fix and format with ESLint
pnpm typecheck    # Type-check with Nuxt
pnpm deploy       # Build + deploy to Cloudflare Workers
```

There are no automated tests in this project.

## Architecture

### Stack
- **Framework:** Nuxt 4 (SSR) with Vue 3 + TypeScript
- **Editor:** TipTap 3 (ProseMirror-based), configured in `app/components/editor/`
- **Styling:** Tailwind CSS 4 + Nuxt UI 6
- **State:** Pinia 3 with `pinia-plugin-persistedstate`
- **Backend:** Nuxt Nitro on Cloudflare Workers
- **Database:** Cloudflare D1 (SQLite) — schema/migrations in `server/utils/db.ts`
- **Storage:** Cloudflare R2 via NuxtHub (`server/utils/r2.ts`)
- **Auth:** HTTP-only session cookies + Google OAuth (`server/utils/auth.ts`)
- **i18n:** Vue I18n 10 (EN + ZH), config auto-generated by `scripts/generateI18n.ts`

### Directory Layout

```
app/
  components/
    editor/          # TipTap editor components and extensions
    documents/       # Document tree, list, share UI
    auth/            # Auth modal
  composables/       # Feature logic (useAI, useDocuments, useAuth, useEditorToolbar, etc.)
  pages/             # File-based routing: index, documents/[id], share/[id]
  store/             # Pinia stores (notification.ts)
server/
  api/               # REST endpoints: auth/, documents/, folders/, ai/, upload
  utils/             # db.ts, auth.ts, r2.ts, ai-log.ts
types/index.ts       # Shared TypeScript types (User, Document, Share, CloudflareEnv)
```

### Key Data Flows

**Editor save:** `RichTextEditor.vue` → `useDocuments.ts` → `PUT /api/documents/[id]` → D1 metadata + R2 content

**AI features:** User selects text → `useAI.ts` → `POST /api/ai/expand|continue|summarize` → Xiaomi AI API → inserted into editor

**Image upload:** `ImageUploadExtension.ts` → `POST /api/upload` → Cloudflare R2 via NuxtHub blob → URL returned to `ImageUploadNode.vue`

**Auth:** `AuthModal.vue` → `useAuth.ts` → `/api/auth/login` → SHA-256 password check against D1 → session cookie → `getCurrentUser(event)` validates on subsequent requests

**Document tree:** `parent_id`-based hierarchy in D1; drag-drop in `DocumentTreeWithDragDrop.vue` calls `/api/documents/[id]/move`

### Editor Extensions

The TipTap editor is extended with: slash commands (`useEditorSuggestions.ts`), drag handle (`useEditorDragHandle.ts`), @ mentions (`useEditorMentions.ts`), emoji picker (`useEditorEmojis.ts`), image upload node, link popover, and Shiki syntax highlighting for code blocks. Toolbar items are defined in `useEditorToolbar.ts`.

### Cloudflare Bindings

- D1 database binding: `sparkles-db`
- R2 bucket binding: `sparkles-r2` (as `BLOB`)
- AI and OAuth credentials are in `runtimeConfig` (from `.env`)

Local development uses `better-sqlite3` and local filesystem blob storage (`.data/blob`) when Cloudflare bindings aren't configured.
